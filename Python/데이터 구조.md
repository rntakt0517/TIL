## Data Structure

컴퓨터 공학에서는 '자료 구조'라고 함

- 문자열, 리스트, 딕셔너리 등 각 데이터 구조의 메서드를 호출하여 다양한 기능을 활용

- **메서드 method** : 'class'라고 하는 객체에 속한 '**함수**'
  
  - 객체의 상태를 조작하거나 동작을 수행
  
  - 특징: 메서드는 클래스(class) 내부에 정의되는 함수
    
    > 클래스는 파이썬에서 '타입을~~~~ 표현하는 방법'
  
  - 각 데이터 타입별로 다양한 기능을 가진 메서드가 존재
  
  - 호출 방법 `데이터 타입 객체.메서드()`
    
    > 메서드는 이어서 사용 가능 (앞쪽의 결과가 맞아야 함)

### 시퀀스 데이터 구조

## 문자열

- **문자열 조회/탐색 및 검증 메서드**
  
  - `s.find(x)` : x의 첫 번째 위치를 반환. 없으면 -1을 반환
  
  - `s.index(x)` : x의 첫 번째 위치를 반환. 없으면 오류 발생
  
  - `s.isupper()` : 대문자 여부
  
  - `s.islower()` : 소문자 여부
  
  - `s.isalpha()` : 알파벳 문자 여부
    
    > 함수 이름 앞에 `is`가 붙으면 반환값이 True나 False

- **문자열 조작 메서드 (새 문자열 반환)**
  
  - <mark>`s.replace(old, new[,count])` : 바꿀 대상 글자를 새로운 글자로 바꿔서 **반환**</mark>
  
  - <mark>`s.strip([chars])` : 공백이나 특정 문자를 제거</mark>
    
    > `[]` 표기는 실제 프로그램에 쓰는 게 아니라 선택 인자임
  
  - <mark>`s.split(sep=None, maxsplit=-1)` : 공백이나 특정 문자를 기준으로 분리하여 문자열의 리스트로 **반환**</mark>
  
  - <mark>`'separator'.join(iterable)` : 구분자로 iterable의 문자열을 연결한 문자열의 **반환**</mark>
  
  - `s.capitalize()` : 가장 첫 번째 글자를 대문자로 변경
  
  - `s.title()` : 띄어쓰기 기준을 각 단어의 첫 글자는 대문자, 나머지는 소문자로 변환
  
  - `s.upper()` : 모두 대문자로 변경
  
  - `s.lower()` : 모두 소문자로 변경
  
  - `s.swapcase()` : 대 <-> 소문자 서로 변환

## 리스트

- **리스트 값 추가 및 삭제 메서드**
  
  - <mark>`L.append(x)` : 마지막에 항목 x를 추가</mark>
    
    > 원본을 바꾸기 때문에 반환 값이 없음 None
  
  - <mark>`L.extend(m)` : Iterable m의 모든 항목들을 끝에 추가 (+=과 같은 기능)</mark>
    
    > 반복 가능한 객체 (리스트) 가 들어가야 됨. 정수X
  
  - `L.insert(i, x)` : 인덱스 i에 항목 x를 삽입
  
  - `L.remove(x)` : 처음 만나는 x를 제거. 항목이 존재하지 않을 경우 ValueError
  
  - <mark>`L.pop([i])` : 마지막(또는 인덱스 i)을 **반환** 후 제거</mark>
  
  - `L.clear()` : 모든 항목 삭제

- **리스트 탐색 및 정렬 메서드**
  
  - `L.index(x)` : 처음 만나는 x의 인덱스를 반환
  
  - `L.count(x)` : x의 개수를 반환
  
  - <mark>`L.reverse()` : 역순으로 변경 (정렬X, 반환X)</mark>
  
  - <mark>`L.sort()` : 오름차순으로 정렬 (매개변수 이용가능)</mark>
    
    > `L.sort(reverse=True)` 은 내림차순

- 문자열에 포함된 문자들의 유형을 판별하는 메서드
  
  - `isdecimal()` : 문자열이 모두 숫자 문자(0~9)로만(**엄격**) 이루어져 있어야 True
  
  - `isdigit()` : 위와 비슷하지만 중간 단계로 **지수** 같은 유니코드 숫자도 인식

- `isnumeric()` : **느슨**한 몇 가지 추가적인 유니코드 문자들(분수, 지수 루트 기호)
  
  > 소수점이나 음수 기호는 셋다 무조건 False

## 복사

- 변경 가능한 데이터 타입의 복사 ex) 리스트

- 변경 불가능한 데이터 타입 ex) 정수

- 복사 유형
  
  - 할당 : 해당 객체에 대한 **객체 참조(방 주소)를 복사**
  
  - **얕은 복사** `b = a[::]` : `L.copy()`랑 동일
    
    > 1단계의 레벨까지만 복사함. 중첩된 리스트일 때 문제 발생
  
  - **깊은 복사** : `import copy` 내장 모듈 활용 `copy.deepcopy(a)`

### 비시컨스 데이터 구조

## 딕셔너리

dictionary : 고유한 항목들의 정렬되지 않은 컬렉션

- 딕셔너리 메서드
  
  - `D.clear()` : 모든 키/값 쌍을 제거
  
  - <mark>`D.get(k[,default])` : 키 k에 연결된 값을 반환 (키가 없으면 None이나 default를 반환)</mark>
  
  - <mark>`D.keys()` : **키**를 모은 객체(반복 가능 형태)를 반환 </mark>
  
  - <mark>`D.values()` : **값**을 모은 객체를 반환</mark>
  
  - <mark>`D.items()` : **키/값 쌍**을 모은 객체를 반환 (`for key, value`로 언패킹하여 활용)</mark>
  
  - <mark>`D.pop(k[,default])` : 키 k를 제거하고 연결됐던 값을 반환 (없으면 오류나거나 default를 반환)</mark>
    
    > 리스트나 딕셔너리의 pop들은 모두 반환을 함
  
  - `D.setdefault(k[,default])` : 키 k와 연결된 값을 반환 (키가 없다면 default와 연결된 키를 딕셔너리에 추가하고 default를 반환)
  
  - `D.update(other)` : other 내 각 키에 대해 D에 있는 키면 D에 있는 그 키의 값을 other에 있는 값으로 대체 (other에 있는 각 키에 대해 D에 없는 키면 키/값 쌍을 D에 추가)

## 세트

set : 고유한 항목들의 정렬되지 않은 컬렉션

- 세트 메서드
  
  - <mark>`s.add(x)` : x를 추가. 이미 x가 있다면 변화 없음</mark>
  
  - `s.clear()` : 모든 항목을 제거
  
  - <mark>`s.remove(x)` : x를 제거. 없다면 Key error</mark>
  
  - `s.pop()` : 임의의 항목을 반환하고, 해당 항목을 제거 **(해시 테이블에 나타나는 순서대로 반환하는 것)**
    
    > "임의"라는 것이 "무작위"는 아님
  
  - `s.discard(x)` : x를 제거. 없다면 변화 없음
  
  - `s.update(iterable)` : 다른 iterable 요소를 추가

- 세트의 집합 메서드
  
  - `set1.difference(set2)` : set1 - set2
  
  - `set1.intersection(set2)` : set1 & set2
  
  - `set1.issubset(set2)` : set1 <= set2
  
  - `set1.issuperset(set2)` : set1 >= set2
  
  - `set1.union(set2)` : set1 | set 2

## 해시 테이블

해시 함수를 사용하여 변환한 값을 인덱스로 삼아 키와 벨류를 저장하는 자료 구조

- **해시** : 임의의 크기를 가진 데이터(key)를 고정된 크기의 고유한 값(index)로 변환하는 것
  
  - 일종의 지문과 같은 역할
  
  - 데이터를 고유하게 식별

- 해시 함수 : 해시 값을 출력하는 함수

- 이유 : 데이터를 효율적으로 저장하고 검색하기 위해 사용

- 원리 : 키를 해시 함수를 통해 해시 값으로 변환하고, 이 해시 값을 인덱스로 사용하여 데이터를 저장하거나 검색

- 장점 : 데이터 검색이 매우 빠르게 이루어짐

- 효율 때문에 정수는 정수 그대로 항상 해시됨
