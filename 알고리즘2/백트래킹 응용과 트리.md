## 백트래킹 응용

여러 가지 선택지들이 존재하는 상황에서 한 가지 선택 반복하며 목표 상태 도달

- 꽝 노드까지 도달하면 최근의 선택으로 되돌아와서 다시 시작

- 백트래킹과 깊이 우선 탐색과의 차이
  
  - **<mark>Prunning 가지치기</mark>**
  
  - NI 가지의 경우의 수를 가진 문제에 대해 깊이 우선 탐색을 가하면 처리 불가능
  
  - 백트래킹 알고리즘을 적용하면 일반적으로 경우의 수가 줄어들지만
    
    최악의 경우에는 여전지 지수함수 시간을 요함

- 8-Queens 문제
  
  - 퀸 8개를 크기의 체스판 안에 서로를 공격할 수 없도록 배치하는 모든 경우
  
  - 후보는 엄청 많지만 실제 해는 92개뿐
  
  - 같은 행에 위치할 수 없다고 가지치기

- 절차
  
  1. 상태 공간 트리의 깊이 우선 검색을 실시
  
  2. 각 노드가 유망한지를 점검
  
  3. 유망하지 않으면, 그 노드의 부모 노드로 돌아가서 검색을 계속
  
  > 이진 트리로 후보를 사용하느냐 마느냐 선택 가능

## 트리 개요

싸이클이 없는 무향(양방향) 연결 그래프

- 비선형 구조
  
  - 원소들 간에 1:n 관계를 가지는 자료구조
  
  - 원소들 간에 계층관계를 가지는 계층형 자료구조
  
  > root 노드 무엇인지  모름

## 이진 트리

모든 노드들이 최대 2개의 서브 트리를 갖는 특별한 형태의 트리

- 높이가 h인 이진 트리가 가질 수 있는 노드
  
  - 최소 h+1 개
  
  - 최대 2**(h+1) - 1 개

- 종류
  
  - 포화 이진 트리 (Full Binary Tree)
  
  - 완전 이진 트리 (Complete Binary Tree)
  
  - 편향 이진 트리 (Skewed Binary Tree)

- 순회 방법
  
  - 전위 순회 (preorder traversal) : VLR
  
  - 중위 순회 (innorder traversal) : LVR
  
  - 후위 순회 (postorder traversal) : LRV

> 삭제가 발생하는 문제에는 1차원 리스트 쓰지말기

## 이진 탐색 트리

BST : 작으면 왼쪽 크면 오른쪽, 탐색작업을 효율적으로 하기 위한 자료구조

- 내 위치를 찾는 과정 = 높이만큼 시간 발생 : O(log n)

- 삭제 : 자식의 개수

- 최악의 경우 : 한쪽으로 치우진 편향 이진 트리

## 힙 트리

Heap Tree : 키 값이 가장 크거나 작은 노드 찾기 위한 자료구조

- 힙 활용 : **<mark>특별한 큐의 구현, 정렬</mark>**

- 파이썬 `heapq` 모듈
  
  - `heapq.heappush(heap, item)`
    
    - item을 heap에 추가
  
  - `heapq.heappop(heap)`
    
    - heap에서 가장 작은 원소를 pop & 리턴
      
      비어 있는 경우 IndexError가 호출됨. 
  
  - `heapq.heapify(x) `
    
    - 리스트 x를 즉각적으로 heap으로 변환함 (in linear time, *O(N)* )
